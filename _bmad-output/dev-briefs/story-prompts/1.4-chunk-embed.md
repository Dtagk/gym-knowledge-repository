Create yt_kg/chunk.py and yt_kg/embed.py implementing Story 1.4: Time-Aware Chunking and LanceDB Embedding.

--- yt_kg/chunk.py ---
Public API: `chunk(video: dict) -> list[dict]`

Implementation:
- Load data/transcripts/{video_id}.json
- Slide a window over the segments list accumulating text until ~2400 chars, then start a new chunk
  with ~300 char overlap (re-include the last segment(s) that push over the boundary in the next chunk)
- Each chunk dict: {chunk_id: f"{video_id}:{i}", video_id, start: float, end: float, text: str}
- Return list of chunk dicts

--- yt_kg/embed.py ---
Public API: `embed()` with no arguments.

Implementation:
- Call init_db(), query: SELECT * FROM videos WHERE transcribed_at IS NOT NULL AND chunked_at IS NULL
- Load SentenceTransformer("BAAI/bge-small-en-v1.5") on CPU — do NOT pass device="cuda"
- For each video:
  - chunks = chunk(video)  # from .chunk import chunk
  - vectors = model.encode([c["text"] for c in chunks], batch_size=32).tolist()
  - Open (or create) LanceDB table at "data/vectors.lance", table name "chunks"
  - Schema: chunk_id str, video_id str, start float, end float, text str, vector list[float] 384-dim
  - Upsert all chunks (use add with mode="overwrite" keyed on chunk_id, or delete+add)
  - UPDATE videos SET chunked_at=utcnow() WHERE video_id=?
  - On any exception: UPDATE videos SET last_error=str(e), error_stage='chunk'; continue

Create data/vectors.lance/ parent directory if needed.
Imports: sentence_transformers.SentenceTransformer, lancedb, pyarrow, .db (init_db, utcnow), .chunk (chunk).
Do not modify any existing files.

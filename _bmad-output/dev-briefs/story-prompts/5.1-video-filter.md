# Story 5.1: Video Filter / Selection Layer

## Context

This is a Python pipeline that ingests YouTube fitness videos into a knowledge graph.
The repo layout is:
- `yt_kg/` — Python package with one module per pipeline stage
- `scripts/run_pipeline.py` — orchestrator that runs stages in order
- `config/channels.yaml` — channel/playlist/video entries
- `data/jobs.sqlite` — SQLite DB managed by `yt_kg/db.py`

The current `discover` stage inserts every video from a channel into the `videos` SQLite table.
The `download` stage then downloads ALL of them (`WHERE downloaded_at IS NULL`).
This is wasteful — a channel like Jeff Nippard has 500+ videos and we only care about
long-form training content, not vlogs, shorts, or reaction videos.

## Goal

Add a `filter` stage that runs **after discover, before download**.
It fetches lightweight metadata for each un-filtered video and marks irrelevant ones
as skipped in the DB so the rest of the pipeline never touches them.

## Database changes — `yt_kg/db.py`

The `videos` table currently has these columns (already created by `init_db()`):
```
video_id, title, channel_id, url,
downloaded_at, transcribed_at, chunked_at, extracted_at, graphed_at, cited_at,
last_error, error_stage
```

Add two new columns with `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`:
- `skipped INTEGER DEFAULT 0` — 1 = skip this video
- `skip_reason TEXT` — human-readable reason (e.g. "duration < 300s", "title matched exclude pattern")

Call this migration inside `init_db()` after the CREATE TABLE so it's idempotent
(SQLite's `ADD COLUMN` is safe to call repeatedly if the column already exists — wrap in try/except).

## New module — `yt_kg/filter.py`

```python
def filter_videos() -> None:
    ...
```

Logic:
1. Load `channels.yaml` — build a dict keyed by `channel_id` → `filter` config.
2. Query DB: `SELECT video_id, channel_id, url FROM videos WHERE downloaded_at IS NULL AND skipped = 0`
3. For each video, fetch its metadata via yt-dlp (no download — `extract_info` with `download=False`).
   Fields needed: `duration` (seconds, int), `upload_date` (YYYYMMDD string), `view_count` (int), `title` (str).
4. Apply the filter rules from that video's channel config (see schema below).
   If any rule fails, mark `skipped=1` and set `skip_reason` to a short description of which rule failed.
5. If all rules pass, leave `skipped=0` (no update needed).
6. Commit after each video so progress is preserved on interruption.

**Filter config schema** (per channel entry in `channels.yaml`):
```yaml
filter:
  min_duration_seconds: 300        # skip if duration < this (default: no minimum)
  max_duration_seconds: 7200       # skip if duration > this (default: no maximum)
  max_age_days: 730                # skip if upload_date older than this many days (default: no limit)
  min_views: 1000                  # skip if view_count < this (default: no minimum)
  title_include:                   # if set, title must match at least one pattern (case-insensitive substring)
    - "hypertrophy"
    - "program"
    - "technique"
    - "training"
    - "workout"
    - "exercise"
    - "muscle"
    - "strength"
  title_exclude:                   # if set, title must NOT match any pattern (case-insensitive substring)
    - "vlog"
    - "react"
    - "podcast"
    - "shorts"
    - "q&a"
```

All filter fields are optional. Missing fields = no filter applied for that criterion.
A video with no channel filter config passes through unchanged.

**yt-dlp call** should use quiet mode and not download anything:
```python
opts = {"quiet": True, "no_warnings": True, "extract_flat": False}
with yt_dlp.YoutubeDL(opts) as ydl:
    info = ydl.extract_info(url, download=False)
```

If yt-dlp raises an exception for a video, log a warning and skip the video
(mark `skipped=1, skip_reason="metadata fetch failed: <error>"`).

## Acceptance criteria

1. Running `filter_videos()` on a DB with 1 un-downloaded video and no filter config → video stays `skipped=0`.
2. A video shorter than `min_duration_seconds` → `skipped=1`, `skip_reason` contains "duration".
3. A video whose title contains a `title_exclude` word → `skipped=1`, `skip_reason` contains "title".
4. Running `filter_videos()` twice on the same DB → same result, no duplicate updates (idempotent).
5. `run_pipeline.py` runs filter stage between discover and download with no errors on the test video.
6. `yt_kg/download.py` WHERE clause excludes `skipped=1` rows.

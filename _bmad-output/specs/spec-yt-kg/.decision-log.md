## 2026-06-20 — YouTube download authentication: cookies.txt over browser extraction

**Decision:** Use a Netscape-format `cookies.txt` file (exported from Chrome via the "Get cookies.txt LOCALLY" extension) rather than yt-dlp's built-in browser cookie extraction.

**Why:** Chrome 127+ switched to App-Bound Encryption for cookie storage, which breaks yt-dlp's DPAPI-based browser cookie reader (yt-dlp issue #10927). Browser extraction now fails with "Failed to decrypt with DPAPI" on all Windows machines running Chrome 127+.

**Tradeoff:** The `cookies.txt` file is a static export — it will expire when the YouTube session expires (typically weeks to months). When downloads start returning 403, re-export the file from the extension. The file is gitignored and never committed.

**Implementation:** `yt_kg/download.py` checks for `cookies.txt` in the project root and prefers it over `cookiesfrombrowser`; falls back to browser extraction if the file is absent (for non-Chrome setups).

**Decision:** The citation stage should attempt to download the full PDF for any paper where OpenAlex returns a non-null `open_access.oa_url`. Papers without an OA URL are recorded with metadata only; no PDF fetch is attempted.
